Beginner-Friendly Guide

Claude Code
핵심 가이드

설치 후 반드시 알아야 할 프로젝트 구조, 설정법, 그리고 실전 워크플로우.
초보자도 5분이면 이해할 수 있도록 정리했습니다.

아래로 스크롤
Part 1

프로젝트 구조 한눈에 보기

Claude Code 프로젝트는 딱 3가지만 기억하면 됩니다: CLAUDE.md(방향타), .claude/rules/(세부 규칙), .claude/skills/(자동화 매뉴얼).

Project Folder Structure
📁 프로젝트/ — 여러분의 프로젝트 최상위 폴더
   ├── 🧠 CLAUDE.md — 프로젝트 브레인 (AI 작업 지침서)
   ├── 📁 .claude/ — Claude Code 설정 폴더
      ├── 📏 rules/ — 주제별 규칙 모음
         ├── 📄 workflow.md — 작업 흐름 규칙
         └── 📄 design-rules.md — 디자인 관련 규칙
      └── ⚡ skills/ — 자동화 스킬 저장소
          └── 📄 daily-summary.skill.md — 매일 요약 스킬 예시
   └── 📦 src/, pages/, ... — 여러분의 실제 프로젝트 파일들
핵심 포인트

CLAUDE.md 하나만 잘 설정해도 AI 성능이 완전히 달라집니다. 배의 방향타처럼, 출발 시 1도 차이가 목적지를 완전히 바꿉니다.

Part 2

CLAUDE.md — 프로젝트 브레인

Claude Code가 대화를 시작할 때 가장 먼저 읽는 파일입니다. 어떤 기술을 쓰는지, 코드 스타일은 어떤지, 프로젝트 구조는 어떻게 되는지를 여기에 적어둡니다.

무엇을 적어야 하나요?

CLAUDE.md # 프로젝트 개요 기술 스택: Next.js + TypeScript + Tailwind CSS 데이터베이스: Supabase (PostgreSQL) 배포: Vercel # 코드 규칙 - 함수 이름은 동사로 시작할 것 (예: getUserData, createPost) - 테스트 코드는 반드시 작성할 것 - 에러 처리는 try-catch 패턴을 따를 것 - 컴포넌트는 함수형으로만 작성 # 프로젝트 구조 src/ components/ → 재사용 UI 컴포넌트 pages/ → 라우팅 페이지 lib/ → 유틸리티 함수 hooks/ → 커스텀 훅

작성 팁 4가지

1

중요한 규칙은 맨 위에

AI는 문서의 시작과 끝 부분을 가장 잘 기억합니다 (Primacy Bias). 핵심 규칙을 파일 맨 위에 배치하세요.

2

200~500줄로 유지

너무 길면 AI가 중간 내용을 놓칩니다. 핵심만 간결하게 적으세요. API 문서 전체를 복붙하면 토큰만 낭비됩니다.

3

실수가 반복될 때 규칙 추가

처음부터 모든 규칙을 적으려 하지 말고, Claude가 같은 실수를 2~3번 반복하면 그때 규칙을 추가하세요.

4

주기적으로 정리

CLAUDE.md도 기술 부채가 쌓일 수 있습니다. 불필요한 규칙은 주기적으로 정리해주세요.

/init 명령어로 자동 생성

Claude Code에서 /init을 입력하면 프로젝트 폴더를 자동 분석하여 CLAUDE.md를 생성해줍니다. 이걸 기반으로 나만의 규칙을 추가하면 됩니다.

3단계 레이어 구조

🌐

Global CLAUDE.md

~/.claude/CLAUDE.md에 위치. 모든 프로젝트에 공통 적용되는 개인 작업 스타일과 선호 규칙을 여기에 적습니다.

📁

Project CLAUDE.md

각 프로젝트 폴더 최상위에 위치. 해당 프로젝트에만 적용되는 기술 스택, 구조, 규칙을 적습니다.

🏢

Enterprise Layer

기업용 Claude에서만 사용 가능한 관리자 레이어. 세 레이어가 병합되어 최종 지침이 완성됩니다.

Part 3

.claude/rules/ — 규칙 모듈화

CLAUDE.md가 점점 비대해질 때 사용합니다. 규칙을 주제별로 분리해서 관리하면 AI가 필요한 규칙만 효율적으로 참조할 수 있습니다.

Before — 규칙이 한 파일에 뒤죽박죽
  • CLAUDE.md가 1000줄 이상
  • 워크플로우, 디자인, 테스트 규칙 섞여있음
  • AI가 중간 내용을 자주 놓침
  • 규칙 수정 시 전체 파일을 뒤져야 함
After — 주제별 파일로 분리
  • 각 파일이 50~100줄로 간결
  • workflow.md, design-rules.md 등 명확한 분류
  • AI가 필요한 규칙만 정확히 참조
  • 규칙 수정/추가가 쉬움

예시: workflow.md

.claude/rules/workflow.md # 작업 흐름 규칙 1. 새 기능 추가 시 - 반드시 플랜 모드로 시작할 것 - 테스트 코드를 먼저 작성할 것 (TDD) - PR 올리기 전 코드 리뷰 에이전트 실행 2. 버그 수정 시 - 재현 테스트를 먼저 작성 - 수정 후 관련 테스트 전체 실행 - 커밋 메시지에 이슈 번호 포함

예시: design-rules.md

.claude/rules/design-rules.md # 디자인 규칙 색상: Tailwind CSS 기본 팔레트만 사용 폰트: Inter (영문), Noto Sans KR (한글) 반응형: 모바일 퍼스트로 작성 컴포넌트: shadcn/ui 기반으로 통일 애니메이션: Framer Motion 사용, 200ms 이하
비개발자를 위한 비유

CLAUDE.md가 회사의 "직원 핸드북" 전체라면, rules/ 폴더는 부서별 업무 매뉴얼입니다. 영업부 매뉴얼, 디자인팀 매뉴얼처럼 주제별로 나눠 관리하는 것과 같은 개념입니다.

Part 4

.claude/skills/ — 자동화 매뉴얼

반복 작업을 단계별 체크리스트로 만들어 저장하면, /스킬이름 한 번에 실행됩니다. 30분 걸리던 작업이 90초로 단축될 수 있습니다.

스킬 파일 구조

.claude/skills/daily-summary.skill.md --- name: daily-summary description: 오늘의 작업 내용을 요약합니다 --- # 실행 단계 1. git log에서 오늘 커밋 내역을 가져온다 2. 변경된 파일 목록을 정리한다 3. 주요 변경사항을 3줄로 요약한다 4. 내일 해야 할 작업을 도출한다 5. 결과를 마크다운 형식으로 출력한다
토큰 절약의 비결

스킬은 호출 전까지 이름과 설명만 로드됩니다 (약 60 토큰). 스킬을 100개 만들어도 컨텍스트를 거의 차지하지 않습니다. MCP 도구 하나가 수천 토큰을 차지하는 것과 대조적입니다.

활용 사례

📧

이메일 분류

수백 개 이메일을 자동 분류하고 중요도별 라벨을 붙이고 답장 초안까지 작성합니다.

🔍

리서치 자동화

특정 주제의 자료를 자동으로 검색하고 핵심만 요약합니다. 수동 30분 작업을 90초로 단축합니다.

🌐

웹사이트 생성

고객별 맞춤 웹사이트를 30초 만에 생성합니다. 템플릿 기반으로 빠르게 커스터마이징합니다.

📊

리드 수집

특정 업종의 비즈니스 연락처를 자동 수집합니다. 영업 담당자 하루 작업을 90초에 처리합니다.

스킬 개선 루프

처음 만든 스킬의 정확도는 약 70%입니다. 테스트하며 실패 부분을 수정하고 2~3번 반복하면 90% 이상으로 올라갑니다. 새 스킬은 반드시 새 세션에서 테스트하세요 (기존 대화 컨텍스트가 결과에 영향을 줄 수 있습니다).

Part 5

실전 워크플로우

Claude Code를 실전에서 제대로 활용하는 두 가지 핵심: Plan Mode와 Task-to-Verify 루프.

Plan Mode — 짓기 전에 설계도 먼저

플랜 모드에서 Claude는 코드를 직접 수정하지 않습니다. 대신 프로젝트 구조를 읽고, 질문이 있으면 물어보고, 어떻게 구현할지 상세한 계획을 먼저 세웁니다. 승인하면 그때서야 실제 코드를 작성합니다.

Plan 없이 바로 빌드
  • 방향을 잘못 잡으면 처음부터 재작업
  • 기능 간 충돌 자주 발생
  • 35분 이상 소요
  • 내비 없이 운전하는 것과 같음
Plan Mode 사용
  • 구현 전 빠진 부분 미리 확인
  • 기능 간 호환성 사전 설계
  • 15분이면 완료 (시간 절반)
  • 계획 1분 = 빌드 10분 절약

Task-to-Verify 루프 — 매번 검증하기

T

Task — 작업 지시

Claude에게 구체적으로 무엇을 만들지 지시합니다. "로그인 페이지를 만들어줘", "이 스크린샷처럼 만들어줘" 등.

E

Execute — AI가 실행

Claude가 코드를 작성하고 파일을 수정합니다. 이 단계에서 AI의 속도가 빛을 발합니다.

V

Verify — 결과 검증

결과물을 확인합니다. 디자인이면 스크린샷을 찍어 비교하고, 코드면 테스트를 실행합니다. 이 단계를 빼먹으면 오류가 누적됩니다.

"AI의 진짜 가치는 완벽함이 아니라 속도에 있습니다. 80% 완성도에서 시작해서 빠르게 수렴하는 거죠."

Context Rot 주의

검증 없이 계속 새 기능을 추가하면, 30분 후 갑자기 아무것도 동작하지 않는 현상이 발생합니다. 오류가 쌓이면서 AI도 혼란에 빠지는 것이죠. 귀찮아도 매 단계 검증하는 습관이 가장 빠른 길입니다.

Part 6

컨텍스트 관리

Claude의 기억 용량(컨텍스트 윈도우)은 20만 토큰입니다. 하지만 첫 메시지 전에 이미 25%가 소비되어 있습니다. 이 연료를 얼마나 효율적으로 쓰느냐가 실력의 차이입니다.

자주 쓰는 슬래시 명령어

명령어기능
/initCLAUDE.md 자동 생성 (프로젝트 분석)
/context현재 컨텍스트 사용량 확인
/compact이전 대화를 고밀도 요약으로 압축
/clear대화를 완전히 초기화
/cost현재까지 사용한 토큰 비용 확인
/model대화 중간에 모델 변경 (Sonnet ↔ Opus)

토큰 절약 전략

  • 프롬프트는 짧고 구체적으로. 길게 설명하는 대신 핵심만 전달하면 토큰을 아낍니다.
  • 서브 에이전트 활용. 리서치 같은 작업을 별도 에이전트에게 위임하면 메인 컨텍스트가 깨끗하게 유지됩니다.
  • 모델 적재적소 배치. 단순 작업에는 Sonnet (저렴), 복잡한 설계에는 Opus (고성능)를 사용합니다.
  • 확장 사고 기능. 복잡한 문제일수록 AI가 깊이 생각하게 하되, 사고 과정의 토큰은 컨텍스트에 누적되지 않습니다.
  • 대화가 길어지면 /compact. 월요일 아침 회의에서 금요일 내용을 못 기억하듯, 컨텍스트가 차면 Claude도 초반 대화를 잊습니다.
Part 7

고급 기능

MCP로 외부 서비스를 연결하고, 서브 에이전트로 병렬 작업하고, 에이전트 팀으로 대규모 프로젝트를 처리합니다.

🔌

MCP (Model Context Protocol)

Gmail, Slack, Notion, Google Sheets 같은 외부 서비스를 Claude Code에 연결하는 플러그인. 설치는 JSON 설정 몇 줄이면 끝납니다. 단, 도구 하나당 수천 토큰을 소비하므로 프로토타입용으로 사용하고, 운영에는 스킬로 변환하세요.

🤖

서브 에이전트

메인 Claude가 특정 업무를 별도 에이전트에게 위임합니다. 각자의 컨텍스트에서 작업하고 결과 요약만 돌려주므로 메인이 오염되지 않습니다. 리서치, 코드 리뷰, QA 테스트에 활용합니다.

👥

에이전트 팀

서브 에이전트의 진화형. 팀원끼리 서로 소통하며 협업합니다. 보안 감사, 대규모 리팩토링에 효과적이지만, 일반 세션 대비 7배 토큰을 소비하므로 중요 작업에만 사용하세요.

⚡

병렬 작업

IDE에서 탭 3~4개를 열어 각각 다른 작업을 동시에 진행합니다. 마치 불평 없는 인턴 3명을 관리하는 느낌. Hook 기능으로 작업 완료 시 알림 소리를 낼 수도 있습니다.

비용 주의

에이전트 팀 10명을 15분 실행하면 약 8만 원이 소요될 수 있습니다. 보안 감사, 대규모 코드 리팩토링 같은 중요 작업에만 사용하세요. 일반 작업에는 서브 에이전트 2~3개면 충분합니다.

오늘 당장 해보세요

지금 바로 실행할 수 있는 3가지 액션 아이템입니다.

1

CLAUDE.md 만들기

/init 명령어로 시작하세요. 거기에 자주 하는 실수나 반드시 따라야 할 규칙을 3줄만 추가해보세요. 이것만으로도 체감 성능이 달라집니다.

2

다음 작업에 Plan Mode 사용

바로 빌드하고 싶은 충동을 참고 계획을 먼저 세우세요. 처음에는 느리게 느껴지지만 전체 작업 시간은 오히려 줄어듭니다.

3

반복 작업 하나를 Skill로 만들기

완벽하지 않아도 됩니다. 70% 정확도로 시작해서 점점 개선하면 됩니다. AI에게 하려는 작업을 설명하면 스킬 파일을 자동 생성해줍니다.

"Claude Code의 성능은 코드를 입력하기 전에 이미 결정됩니다. 설정에 5분 투자하면 이후 수십 시간을 아낄 수 있습니다."

기억하세요

코딩은 Claude Code가 합니다. 여러분은 한국어로 무엇을 만들지 말하고, 결과를 확인하고, 피드백을 주는 역할만 하면 됩니다. 프로그래밍 지식이 없어도 충분히 시작할 수 있습니다.
