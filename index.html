Agent Teams (멀티 에이전트)

HOT

여러 AI가 팀으로 협업

하나의 세션이 "팀 리드"가 되어, 여러 Claude 인스턴스가 공유 태스크 리스트를 기반으로 협업합니다. 서브에이전트와 달리 팀원끼리 직접 메시지를 주고받을 수 있습니다.

주요 특징

• 팀 리드가 작업 조율
• 팀원 각각 독립 컨텍스트
• 팀원 간 직접 통신 가능
• 공유 태스크 리스트 기반

생명주기

TeamCreate → TaskCreate x N
→ Task(team, name) 스폰
→ 팀원들이 태스크 수행
→ SendMessage(shutdown)
→ TeamDelete
💡 Shift+Down으로 팀원 간 이동, Ctrl+F로 백그라운드 에이전트 전체 종료
🤖

커스텀 서브에이전트

CORE

전문 에이전트를 직접 만들어 사용

AGENTS.md 파일이나 .claude/agents/ 폴더에 마크다운으로 에이전트를 정의하면, Claude가 자동으로 인식하고 사용합니다.

# .claude/agents/security-reviewer.md

---
name: security-reviewer
description: 보안 취약점 전문 리뷰어
model: sonnet              # 모델 선택 가능
tools: [Read, Grep, Glob]  # 사용 가능 도구 제한
---

보안 리뷰어입니다. OWASP Top 10 기준으로 코드를 검토합니다...

포그라운드 vs 백그라운드

• 포그라운드: 결과를 기다린 후 진행
• 백그라운드: 독립적으로 실행, 완료 시 알림
• 백그라운드 종료 시 부분 결과 보존

모델 라우팅

태스크 복잡도에 따라 모델 선택:
• haiku — 빠른 조회, 간단한 검색
• sonnet — 일반 구현, 리뷰
• opus — 아키텍처, 복잡한 분석

Hooks (자동화 훅)

HOT

특정 이벤트에 자동으로 코드 실행

settings.json에 훅을 등록하면, Claude의 동작 전후에 자동으로 스크립트가 실행됩니다. CI/CD, 린트, 알림 등을 자동화할 수 있어요.

훅 이벤트발동 시점활용 예시
PreToolUse도구 실행 전위험한 명령 차단
PostToolUse도구 실행 후자동 린트/포맷
SessionStart세션 시작환경 초기화, 컨텍스트 주입
SessionEnd세션 종료정리 작업, 로그 저장
UserPromptSubmit사용자 입력 제출입력 전처리
SubagentStart/Stop서브에이전트 시작/종료에이전트 모니터링
WorktreeCreate/Remove워크트리 생성/삭제VCS 설정/정리
TaskCompleted태스크 완료알림 전송
PreCompact컨텍스트 압축 전중요 정보 보존
ConfigChange설정 변경설정 감사 로그
💡 HTTP 훅도 지원 (type: "http"): 외부 서비스에 웹훅으로 이벤트를 전달할 수 있습니다

에이전트마다 독립된 작업 공간

여러 에이전트가 동시에 코드를 수정할 때 발생하는 머지 충돌 문제를 근본적으로 해결합니다. 각 에이전트가 독립된 브랜치와 디렉토리에서 작업합니다.

Agent(
  prompt="인증 모듈 리팩토링",
  isolation="worktree"    # 독립 워크트리에서 실행
)

# 작업 완료 후 변경사항이 있으면 브랜치+경로 반환
# 변경사항 없으면 자동 정리

Agent SDK

세션 관리대화 이력 유지, 세션 재개

실전 활용 팁

1. CLAUDE.md로 프로젝트 규칙 설정

프로젝트 루트에 CLAUDE.md를 만들면 Claude가 자동으로 읽고 규칙을 따릅니다.

# CLAUDE.md 예시
- Python 3.9 호환 유지
- 테스트: pytest (venv 활성화 필요)
- 커밋 전 lint 실행 필수
- 3개 이상 파일 수정 시 계획 먼저 작성

2. 멀티 에이전트로 병렬 작업

독립적인 작업을 여러 에이전트에 분배하면 작업 속도가 크게 향상됩니다.

# 3개 에이전트가 병렬로 작업
Agent(prompt="API 엔드포인트 구현", isolation="worktree")
Agent(prompt="테스트 코드 작성", isolation="worktree")
Agent(prompt="문서 업데이트", isolation="worktree")

3. Hooks로 자동화 파이프라인 구축

파일 저장 후 자동 린트, 커밋 전 검증 등을 hooks로 자동화하세요.

// settings.json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Edit|Write",
      "command": "npx prettier --write $FILE"
    }]
  }
}

4. /loop로 반복 모니터링

배포 상태, 빌드 결과 등을 주기적으로 자동 체크할 수 있습니다.

/loop 5m 배포 상태 확인해줘        # 5분마다 확인
/loop 10m gh run list --limit 3  # 10분마다 GitHub Actions 확인
